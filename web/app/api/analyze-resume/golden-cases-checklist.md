# Golden Cases Checklist

Используйте этот чек-лист при ручной проверке `POST /api/analyze-resume` на кейсах из `golden-cases.json`.

## Базовая валидность

- Ответ валидный JSON, без markdown-обертки.
- `score` в диапазоне `0..100`.
- Есть секции: `skills`, `experience`, `fit`, `structure`.
- `topActions` содержит от 1 до 3 пунктов.

## Качество комментариев

- Для каждого `problem` есть `evidence` или `questionToCandidate`.
- `whyItMatters` объясняет влияние на ATS/скрининг, а не повторяет описание.
- `rewrite` не выдумывает факты, которых нет в резюме.
- `confidence` понижается до `low/medium`, если вывод основан на неполных данных.
- `impact` помогает приоритизировать правки (есть хотя бы 1 `high` для слабого резюме).

## Анти-галлюцинации

- Нет технологий, сертификатов или опыта, которых нет в тексте резюме/вакансии.
- Если данных недостаточно, модель задает уточняющий вопрос в `questionToCandidate`.
- Сильные стороны (`kind: strength`) не маскируются как проблемы.
